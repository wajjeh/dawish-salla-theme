# Launch Checklist — عطارة الدويش

## قبل المعاينة
- [ ] `twilight.json` في جذر GitHub مباشرة.
- [ ] `package.json` في الجذر.
- [ ] مجلد `src` في الجذر.
- [ ] اختيار فرع `main` في لوحة الثيم.
- [ ] ربط TwilightCI على نفس حساب GitHub ومع صلاحية المستودع.

## إعدادات الثيم في سلة
- [ ] تفعيل شاشة التحميل.
- [ ] تفعيل البحث الكبير.
- [ ] تفعيل الهيدر المثبت.
- [ ] تفعيل المفضلة في الهيدر.
- [ ] تفعيل أزرار واتساب والرجوع للأعلى.
- [ ] ضبط رقم واتساب بدون + إذا رغبت.
- [ ] ضبط نص الشريط العلوي.

## اختبار الواجهة
- [ ] الصفحة الرئيسية تظهر بدون أخطاء.
- [ ] البحث يفتح ويغلق.
- [ ] القائمة الجانبية بالجوال تفتح وتغلق.
- [ ] السلة والحساب يعملان.
- [ ] صفحة المنتج تعرض الصور والسعر والإضافة للسلة.
- [ ] صفحة السلة تعمل.
- [ ] الفوتر يظهر روابط التواصل والسياسات.

## اختبار الجوال
- [ ] الهيدر لا يغطي المحتوى.
- [ ] المنتجات تظهر كرتين بالصف أو بشكل واضح حسب مساحة الشاشة.
- [ ] زر واتساب لا يغطي زر إضافة للسلة.
- [ ] زر الشراء الثابت في صفحة المنتج يعمل.

## قبل النشر
- [ ] تجربة المتجر التجريبي من سلة.
- [ ] تجربة الطلب حتى السلة.
- [ ] التأكد من صور المنتجات والأقسام.
- [ ] التأكد من صفحات السياسات.
- [ ] التأكد من بيانات التواصل.

## فحص تحديث الأحجام والجوال

- [ ] خيارات الأحجام تظهر كبطاقات واضحة وليست صور منتجات مزعجة.
- [ ] عند اختيار حجم يظهر إطار زيتوني وعلامة اختيار.
- [ ] رسالة “تم اختيار” تتحدث حسب الحجم المختار.
- [ ] زر إضافة للسلة الثابت لا يغطي شريط التنقل السفلي.
- [ ] زر واتساب لا يغطي زر إضافة للسلة.
- [ ] السلة تعرض المنتج والكمية بدون تداخل على الجوال.
- [ ] حقل كوبون الخصم لا يكسر تصميم السلة على الجوال.
- [ ] صفحة إتمام الطلب قابلة للاستخدام بدون تداخل مع عناصر عائمة.
